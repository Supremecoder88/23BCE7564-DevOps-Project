## 3. Implementation Workflow

### Step 1: Version Control (Git)
- The source code (HTML, CSS, JS) is stored in a Git repository.
- **Key Files**: `index.html`, `style.css`, `script.js`.

### Step 2: Containerization (Docker)
- A `Dockerfile` was created to package the website into a lightweight Nginx container.
- **Command**: `docker build -t techsymp-website .`

### Step 3: CI/CD Pipeline (Jenkins)
- A `Jenkinsfile` defines the automated pipeline:
    - **Checkout**: Pull code from GitHub.
    - **Build**: Create Docker image.
    - **Push**: Upload image to Docker Hub.
    - **Deploy**: Update Kubernetes deployment.

### Step 4: Orchestration (Kubernetes)
- `k8s/manifests.yaml` defines a **Deployment** with 3 replicas for high availability and a **NodePort Service** to expose the website on port `30001`.

### Step 5: Continuous Monitoring
- **Nagios**: Configured to monitor HTTP status 200 on port 30001.
- **Graphite**: Collects system metrics (CPU, Memory).
- **Grafana**: Visualizes metrics through custom dashboards.
